前言：架站…究竟架在哪兒好？

在網際網路上居無定所了多年，到處漂流。從 Apache HTTP server 到簡單的單頁式 Github Page，後來投入好像很多人在用的 Wordpress，然而最後還是飄到了 Hexo 這裡。

簡單評註一下，供參考：

  • Apache：可以從底層建構一個伺服器，很好玩，但很折騰。
  • 單頁式靜態網頁：很適合當做個人形象頁面，方便管理，也有很多免費的實踐方法。
  • Wordpress：想寫部落格又想自己當主人（痞客邦之類的真的很醜欸），就非常適合。但缺點是，幾乎不可能免費做到。
  • Hexo + Github Page：想要兼併靜態網頁的簡單美好，又想稍微寫寫部落格，分享一些內容？那這就是一個不錯的選擇。

其實 Wordpress 蠻好用的，很多 Plug-in 可以玩轉。但用了一陣子後，沒想到他比想象中要吃資源。我的 Wordpress 部落格幾乎沒有人造訪，也沒放多少文章上去，但在 GCP 上租的主機就每個月吃掉我 200NTD 以上，利弊衡量之下實在是無法接受，就果斷砍掉了。

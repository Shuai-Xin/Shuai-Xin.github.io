音樂製作：行情參考價

發表於2022-08-20|更新於2025-05-25|數位音樂製作
|瀏覽量:

前言

現在因為「數位音樂製作」的崛起，消費級的軟硬體設備大行其道，「宅錄」已經不是什麼新鮮事。但「宅錄」這個名詞我個人不太喜歡，聽起來就有點「業餘」——畢竟，有很多經典的作品其實是「宅錄」誕生的。我會說，現在音樂製作這塊領域，已經有一部分天地是「個人工業」的版圖了。

然而生產者一多，也導致了難以避免的「削價競爭」。很多初學製作不久的學生也會開始在社群上自主接案，開價跨幅之大，時而令人瞠目、時而令人咋舌。在這樣百花齊放的時代，行情價更是令人難以掌握。

故，在這邊簡單的就我自己在業內的經驗，列出一些行情價碼，供大家參考。撰寫當下為西元年 2022，幣值單位為新台幣。

參考行情價

  • 詞：10000 起跳，其後續版稅另計。如買斷則價更高。
  • 曲：10000 起跳，其後續版稅另計。如買斷則價更高。
  • 編曲：30000 起跳，其後續版稅視契約而定，不過目前編曲較少在算版稅。
  • 錄音：2000/hr 起跳。錄音階段通常是以鐘點計時，錄音室器材與錄音師包在一起算。
  • 和聲配唱：8000 起跳。
  • 人聲後期：視段落多寡而定，鐘點大約與錄音師近似。
  • 混音：10000 起跳。
  • 母帶製作：5000 起跳。

後記

所以說，想拿一兩萬經費製作一首完整的歌曲就是在破壞行情。

認真想做出一首完整編制、內容上道的流行歌曲，十萬跑不掉。

解決 Chrome 的 Localhost Error
問題 架站時總是會先在瀏覽器上預覽效果，這時通常會採用 localhost 來訪問。 但有好一陣子我的 Chrome 就是直接一個： 這個網站無法提供安全連線， localhost 傳送的回應無效。 並且錯誤代碼為 ERR_SSL_PROTOCOL_ERROR。 原因 究其原因，其實是因為HTTP嚴格傳輸安全（HTTP Strict Transport Security, HSTS）發作了。 簡單來說，就是曾經用 https 造訪的網站，在你下次用 http 再次光臨時，瀏覽器會很「貼心地」幫你改成使用 https。而如果此時該網站並不支援 https ，那就會報這個錯誤了。 解決方案 在 Chrome 的網址列輸入 chrome://net-internals/#hsts，往下找到 Delete domain security policies 之處，在 Domain 欄位輸入 localhost，並且 Delete 掉。 噠啦！此時你的 localhost 就脫離了 HSTS 的魔爪了。

在 Mac 上安裝 Home Assistant

發表於2023-12-15|更新於2026-01-24|其他

用什麼來跑 Home Assistant 呢？

為了把小米的設備整合到 Apple 家庭之內，我也來搗鼓 Home Assistant 了。

本來想買個樹莓派，用 docker 裝 Home Assistant 來玩，好處是同一台樹莓派可以同時拿來玩個 NAS 之類的，而且畢竟作為伺服器必須全年無休，功耗也是樹莓派較為理想。但後來冷靜下來，還是先拿閒置的 2015 MacBook Pro 來玩吧，免得我三分鐘熱度，世界上又將再多出一個被遺忘的算力……（看向被我關在抽屜裡的好幾塊 Arduino）

嗯，如果接下來 Home Assistant 順利融入我的生活，也許可以先淘一塊香橙派當個過渡，後續再視情形升級成樹莓派咯？

Please keep in mind that the bridged adapter only functions over a hardwired Ethernet connection. Using Wi-Fi on your VirtualBox host is unsupported.

心都涼了。因為我的路由器跟房間沒有牽線，Mac 那麼大一台也不方便放在路由器旁邊…… 算了，心一橫，我命由我不由天，咱來硬闖！

運行狀況

目前運行良好，設備反應速度挺快，感受上甚至比米家操作還更流暢快速。

然而，雖然 Mac 端已經設定永遠不進入休眠，但一覺醒來還是會出現伺服器打瞌睡的狀況，目前還不曉得如何處理。

這個打瞌睡就很奇怪，明明沒有休眠，滑鼠動一下就秒醒，到底為啥偷懶呀？
